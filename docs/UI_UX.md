# UI/UX Guidelines

Цель: Создать ощущение нативного мобильного приложения (iOS-like feel) внутри Telegram Web App.

## Дизайн-система

- **Цвета:** Используются CSS переменные (`--background`, `--primary`).
  - Dark Mode по умолчанию: `#09090b` (Zinc 950).
  - Акцентный цвет: `#38bdf8` (Sky 400) или кастомный `themeColor` из API магазина.
- **Типографика:** Системные шрифты (`-apple-system`, `Roboto`). Крупные заголовки, четкие цены.
- **Safe Areas:** Учитываем нижнюю зону (Home Indicator на iPhone) с помощью паддингов `pb-safe` (или ручных отступов), чтобы кнопки не перекрывались системными элементами.

## Компоненты

### Drawers (Шторки)
Мы используем библиотеку **Vaul** для всех модальных окон.
- Она эмулирует нативное поведение iOS (drag to close, backdrop blur).
- Пример: `ProductDrawer`, `CheckoutDrawer`.

### Sticky Header
Сложный компонент в `widgets/layout`.
- Использует **Framer Motion** (`useScroll`, `useTransform`).
- Анимация: При скролле большая обложка превращается в компактный навбар с названием заведения.
- Категории "прилипают" к низу хедера.

### Анимации
Используем `framer-motion` для микро-взаимодействий:
- `active:scale-[0.98]` на кнопках (тактильный отклик).
- `layout` prop для плавной перестройки списков.
- `AnimatePresence` для появления/исчезновения элементов (например, `CartSummaryBar`).

## Работа с изображениями

- Все изображения должны иметь соотношение сторон, предусмотренное дизайном (обычно 1:1 или 4:3).
- В `ProductCard` используется градиент поверх изображения, чтобы белый текст читался на любом фоне.

## Форматирование цен

- **Важно:** Все цены в коде хранятся в минимальных единицах (копейках).
- При выводе делим на 100: `(price / 100).toFixed(0)`.
